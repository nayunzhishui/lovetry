# UI 方向探索与结论

## A：双人年鉴 Twin Almanac（采用）

- 结构：首页像当天年鉴页，关系状态置顶，今日安排与最近记录沿双轨向下；
- 主操作：底部固定四入口，页面内只保留一个主按钮；
- 组件：日期铅字、双轨状态、档案卡、印章标签、低饱和状态色；
- 优化：长期记录感、共享/私密辨识、核心入口效率；
- 牺牲：不追求传统粉色“甜宠感”；
- 采用理由：直接延续现有双轨资产，又与 GitHub 同类项目形成明显差异。

## B：夜航星图 Quiet Observatory

- 结构：深色全屏、纪念日与计划像星点分布；
- 主操作：中心浮动记录按钮；
- 优化：沉浸、纪念感和夜间使用；
- 牺牲：日间可读性、复杂表单和相册展示效率。

## C：记忆拼贴 Memory Tiles

- 结构：照片、便签、票根式自由卡片拼贴；
- 主操作：右下角悬浮创建；
- 优化：相册和共同动态的情绪价值；
- 牺牲：任务、账本、搜索和长列表的一致性。

## 采用方向的美术系统

### 色彩

- 雾纸 `#F1F4F2`：全局背景；
- 档案白 `#FCFDFC`：卡片与表单；
- 铅墨 `#1F2D31`：主文字；
- 钴蓝 `#35658D`：第一条人物轨道/信息状态；
- 珊瑚 `#D46557`：第二条人物轨道/关键动作；
- 苔绿 `#738E78`：共享完成状态；
- 黄铜 `#C49A4A`：纪念日与积分。

### 字体

- 展示数字/标题：`Songti SC, STSong, serif`，只用于天数、月份和一级标题；
- 正文：`-apple-system, BlinkMacSystemFont, Helvetica Neue, PingFang SC, sans-serif`；
- 时间戳/标签：`Menlo, Consolas, monospace`，小范围使用。

### 布局与签名

- 页面水平安全边距 28rpx，8rpx 基础网格；
- 档案卡圆角 10rpx、按钮圆角 6rpx，避免大面积胶囊；
- 仅用轻边框和短投影，禁止玻璃拟态和大面积渐变；
- 签名元素：两条 4rpx 轨道在共享节点汇合；私密节点只显示本人轨道；
- 动效只保留页面首个节点汇合与按钮按压，遵守减少动效设置。

## P9 行动层级补充（2026-07-18）

- 具体对象：需要在碎片时间快速记录与安排生活的两位伴侣；
- 单一任务：从“今天”直接进入一次记录、安排或复盘；
- 色彩继续使用雾纸、档案白、铅墨、钴蓝、珊瑚与苔绿，不新增主题色；
- 标题继续使用宋体，正文使用系统无衬线，时间和状态使用等宽字体；
- 计划页采用“类型 → 新增入口 → 现有清单”的清单优先结构，展开后才出现表单；
- 首页行动区使用三条短轨道区分自己、两人和沟通，不采用通用悬浮加号；
- 日历事项采用“时间 / 标题 / 中文语义 / 箭头”的档案索引结构；
- 自审调整：舍弃大面积渐变和新增插画，只在 8rpx 宽的“两人行动轨”内发生一次汇合，保持 Twin Almanac 的克制辨识度。

## P11 授权与建议落地补充（2026-07-18）

- 具体对象：希望获得贴近情境的建议，同时不愿让 AI 自动读取私人数据的用户；
- 单一任务：看清将要提供的记录内容，并可以随时不选、移除或只用一次；
- 延续既有雾纸、档案白、钴蓝和珊瑚，不新增 AI 紫色、发光渐变或机器人插画；
- 新签名“双轨授权单”：两条细轨代表“本人记录”和“临时进入 Agent”，选中时只改变边框与内容，不制造高兴奋感；
- AI 建议进入记录表单时使用档案交接单，明确“私密优先、二次确认、不会自动发送”；
- 奖励提案使用黄铜待确认状态，双方同意后转为苔绿，不用红色制造关系压力；
- 自审调整：舍弃全屏授权弹窗和默认加载记录，只在用户主动点击后展开候选，减少隐私焦虑和页面阻塞。
