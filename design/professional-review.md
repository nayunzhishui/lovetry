# Lovetry 多专业综合评价

> 评价日期：2026-07-18
> 范围：产品流程、关系心理安全、工程实现、UI 美术与无障碍
> 边界：这是产品内容与心理安全审查，不构成心理咨询、诊断或治疗意见。

## 总体判断

**需要定向改进。** 产品已经形成稳定的“双人年鉴”辨识度，核心数据权限、草稿恢复、错误状态和 AI 安全底线明显高于普通演示项目；目前主要问题不在功能数量，而在可信状态表达、关系机制边界、深色视觉可读性和少量工程冗余。

## 四维评价

### 1. 产品体验：基础扎实，首用引导仍可更具体

优点：今天、记录、日历、计划形成清晰主路径；记录与计划均有草稿和快速模板；空状态能引导下一步。

问题：恋爱助手首屏先展示模型连接实现细节，遮挡真实价值；开放输入仍要求用户自己组织复杂关系情境；AI 成功降级到知识库时使用红色错误样式，容易误解为回答失效。

### 2. 关系心理安全：方向正确，奖励机制需要更强边界

优点：沟通复盘区分事实、感受、需要与约定；敏感记录默认私密；AI 不做人格或疾病诊断，不提供监控、报复或操控建议，并对暴力、自伤与强制控制优先安全响应。

问题：积分与奖励可能被理解为“用表现换取亲密或服从”，目前缺少双方自愿、可以拒绝以及不得交换隐私、身体接触、道歉或控制权的明确提示；模型提示还应避免仅凭单方叙述确认伴侣动机。

### 3. 工程实现：模块边界清楚，存在配置错误与重复路径

优点：云函数、共享领域逻辑、客户端适配层分离；私密权限、幂等、版本冲突、降级和配额均有测试；API 密钥不进入客户端。

问题：`requiredPrivateInfos` 错误声明 `chooseMedia`，微信官方文档限定该字段为位置类接口；旧 `pages/conflict/conflict` 已无入口，与统一记录表单形成重复实现；AI 提供方状态文案散落在页面脚本中。

OpenAI 官方模型目录确认 `gpt-5.6-luna` 是正式的成本敏感模型，因此保留当前默认模型，不做无依据替换。

### 4. UI 美术与无障碍：视觉有辨识度，深色主题未继承语义色

优点：Twin Almanac 的雾纸、档案卡、宋体标题和双轨标记具有一致性；大部分交互使用原生按钮；小屏与减少动效已有基础适配。

问题：深色背景 `#142126` 上，钴蓝 `#35658D` 对比度约 2.66:1、黄铜深色 `#735A2D` 约 2.53:1、危险色 `#B44F45` 约 3.25:1，不适合作为小号状态文字；AI 回答没有把生成模式绑定到每一条消息，来源状态会被后续回答覆盖。

## 高优先级改进

| 优先级 | 问题 | 改进 | 验收 | 结果 |
|---|---|---|---|---|
| P0 | AI 回答来源不可持续识别 | 每条回答显示 AI / 知识库 / 安全模式；降级改为中性通知 | 历史回答保留各自模式 | 已完成 |
| P0 | 奖励机制边界不足 | 增加自愿、可拒绝和不可交换事项说明 | 账户与商城入口可见 | 已完成 |
| P0 | 深色语义色对比不足 | 为深色主题设置独立高对比语义色并增加自动测试 | 关键文字对比度至少 4.5:1 | 已完成 |
| P1 | AI 输入组织负担高 | 提供“梳理沟通”和“准备开口”结构模板 | 一次点击得到可编辑框架 | 已完成 |
| P1 | 微信配置错误 | 移除 `requiredPrivateInfos.chooseMedia`，保留隐私指引说明 | 项目配置校验通过 | 已完成 |
| P1 | 沟通页面重复 | 统一使用支持草稿与可见性控制的记录表单，移除旧页面 | 仅保留一条沟通复盘路径 | 已完成 |

## 实施结果

- 建立 `shared/agent-experience.js`，统一回答来源、连接状态和输入框架表达；
- 增加事实/理解/未知区分、伴侣动机不确定性和性强迫安全规则；
- 奖励账户与商城均展示同意边界；
- 深色关键语义色对比度由自动测试守护，确认弹窗补充对话框语义；
- 清理错误全局配置和重复页面；
- `npm run quality` 通过：70 项测试、17 个页面、9 个云函数、89 个 JavaScript 文件，主包源码 239.6 KiB。

## 暂不自动完成

- 真实微信开发者工具、iOS/Android 深色渲染和屏幕阅读器验证；
- 奖励机制的真实双人可用性访谈；
- 恋爱知识内容的持证专业人员复核；
- 外部模型真实密钥、费用和网络环境验收。
