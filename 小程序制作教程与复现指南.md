# 情侣记录小程序制作教程与复现指南

## 目标

本项目先做一个微信原生小程序 MVP，用云开发同步情侣双方的私密记录。核心功能包括：

- 创建或加入情侣空间
- 记录恋爱经历
- 记录吵架复盘
- 按时间线查看记录
- 数据保存在微信云开发数据库中

## 推荐可复现教程与项目

1. 微信官方小程序示例
   - GitHub: https://github.com/wechat-miniprogram/miniprogram-demo
   - 用途：学习项目结构、页面、组件、API、云开发示例。
   - 复现方式：克隆仓库后运行 `npm run init`，再用微信开发者工具打开，并在工具里执行“构建 npm”。

2. TDesign 微信小程序组件库
   - GitHub: https://github.com/Tencent/tdesign-miniprogram
   - 用途：后续把表单、按钮、弹窗、列表升级成更成熟的 UI。
   - 当前 MVP 暂不引入，避免 npm 构建和组件配置增加复杂度。

3. 腾讯云 CloudBase 小程序快速开始
   - 文档：https://docs.cloudbase.net/quick-start/mini-program/introduce
   - 用途：配置微信云开发环境、数据库、云函数和权限。

## 导入步骤

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择：

```text
D:\codex\workspace\lovetry\couple-miniprogram
```

4. 如果还没有 AppID，可先用测试号或小程序 AppID。
5. 开通云开发，创建一个云开发环境。
6. 打开 `couple-miniprogram/miniprogram/config.js`，把 `envId` 改成你的云开发环境 ID。
7. 在微信开发者工具左侧找到 `cloudfunctions`，分别上传并部署：
   - `login`
   - `couple`
   - `records`
8. 在云开发数据库中创建两个集合：
   - `couples`
   - `records`
9. 开发阶段可先把集合权限设为“仅创建者可读写”或通过云函数读写；正式使用前建议收紧为“所有端不可读写，仅云函数可读写”。

## 数据设计

`couples` 集合保存情侣空间：

```js
{
  code: "6位加入码",
  members: ["openid1", "openid2"],
  createdBy: "openid1",
  createdAt: Date
}
```

`records` 集合保存恋爱事件和吵架记录：

```js
{
  coupleId: "情侣空间 ID",
  type: "moment | conflict",
  title: "标题",
  content: "内容",
  payload: {},
  creatorOpenid: "openid",
  createdAt: Date,
  updatedAt: Date
}
```

## MVP 验收标准

- 能在微信开发者工具中打开首页。
- 能调用 `login` 云函数获取当前用户 `openid`。
- 能创建情侣空间并显示加入码。
- 另一位用户能用加入码加入同一空间。
- 能提交恋爱经历记录。
- 能提交吵架复盘记录。
- 时间线能显示当前情侣空间下的记录。
